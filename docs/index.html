Docs / Introduction

Introduction to MEX

Learn Machine Learning by writing code, not by reading textbooks.

MEX is a beginner-friendly programming language designed to teach machine learning concepts through a clean, readable DSL (Domain Specific Language). It mirrors TensorFlow.js syntax so skills transfer directly to real-world ML development.

💡 Why MEX?

Traditional ML requires complex setup, math-heavy theory, and boilerplate code. MEX strips away the complexity so you can focus on understanding concepts, not debugging imports.

Key Features

🧠

Zero Dependencies

Built-in neural network engine. No npm packages needed.

📖

Readable Syntax

Code reads like English. data points, train, predict.

⚡

5 Lines to ML

Load data, create model, train, predict. That's it.

🔗

TF.js Bridge

See equivalent TensorFlow.js code. Skills transfer directly.

Docs / Installation

Installation

MEX requires Node.js 14+ and has zero external dependencies.

Step 1: Get Node.js

Download from nodejs.org (LTS version recommended).

Step 2: Verify Installation

terminal 
node --version   # Should show v14 or higher

Step 3: Clone or Download MEX

terminal 
git clone https://github.com/your-repo/mex.git
cd mex

Step 4: Test It

terminal 
node mex.js examples/01_data.mx
✅ That's it!

No npm install. No native modules. No build step. Just run and learn.

Docs / What is MEX?

What is MEX?

MEX is a programming language designed specifically for learning machine learning. It's not a library, not a framework, not a wrapper — it's a complete language with its own syntax, parser, and interpreter.

Language Features

  • Custom syntax — ML-specific keywords like data points, model, train
  • Variables — let x = 10
  • Functions — fn add(a, b) { return a + b }
  • Loops — for i in range(0, 10) { ... }
  • Conditionals — if (x > 5) { ... } else { ... }
  • Import/Export — Modular code across files
  • Built-in math — sqrt(), pow(), abs(), sin(), etc.

ML Features

  • Neural network engine — Built from scratch, zero dependencies
  • Data normalization — Automatic scaling
  • Training — Gradient descent with configurable epochs
  • Prediction — Use trained models instantly
  • TF.js bridge — See equivalent TensorFlow.js code
Docs / MEX vs Normal Packages

MEX vs Normal Packages

MEX is fundamentally different from normal npm packages. Here's how:

MEX (Learning Language)
# It's a LANGUAGE you learn
# with its own syntax

data points
  (1, 2)
  (2, 4)

model simple
train 100 epochs
predict 5

# You RUN it with:
# node mex.js file.mx
Normal Package (Library)
// It's a LIBRARY you import
// with JavaScript syntax

const tf = require('@tensorflow/tfjs');

const model = tf.sequential();
model.add(tf.layers.dense({
  units: 1,
  activation: 'linear'
}));

// You import it in JS code
Aspect MEX Normal Package
Type Programming Language JavaScript Library
Syntax Custom ML-focused Standard JavaScript
Purpose Teach ML concepts Build production apps
Dependencies Zero Often many
Learning Curve Beginner-friendly Requires JS knowledge
Execution node mex.js file.mx require('package')
Output Educational, clear Performance-optimized
Progression Leads to TensorFlow.js Direct production use

The MEX Path

🎯 Learning Progression

MEX (learn concepts) → TensorFlow.js (build real apps) → Production ML

MEX teaches you how ML works. TensorFlow.js lets you build with ML. Both use the same core concepts.

When to Use MEX

  • Learning machine learning for the first time
  • Understanding how neural networks work
  • Teaching ML to students
  • Prototyping ML concepts quickly

When to Use Normal Packages

  • Building production applications
  • Need maximum performance
  • Integrating with existing JavaScript projects
  • Advanced ML research

# Best for linear patterns

Sequential Model (Neural Network)

mx

# Multiple layers for complex patterns

Layer Types

Layer Syntax Use Case
dense dense <units> <activation> Fully connected layer
dropout dropout <rate> Prevent overfitting

Activation Functions

Activation Range Best For
relu [0, ∞) Hidden layers (most common)
sigmoid (0, 1) Binary classification
softmax (0, 1) sums to 1 Multi-class classification
tanh (-1, 1) Hidden layers
linear (-∞, ∞) Regression
Docs / Training

Training

Training teaches the model to find patterns in your data.

Basic Training

mx 
train 100 epochs

# 1 epoch = model sees all data once
# More epochs = more learning

With Optimizer

mx 
train with adam
loss cross_entropy
epochs 100

What Happens During Training

  1. Forward pass — Data flows through the model
  2. Calculate error — How wrong was the prediction?
  3. Backward pass — Adjust weights to reduce error
  4. Repeat — Do this for N epochs

Understanding Error

📊 Error Values

Lower error = better model

  • Error near 0 — Model learned well
  • Error still high — Need more epochs or different data

  }
  return n * factorial(n - 1)
}

Docs / Import/Export

Import/Export

Organize code across multiple files.

Exporting

lib/math.mx 
export fn add(a, b) {
  return a + b
}

export fn multiply(a, b) {
  return a * b
}

export let pi = 3.14159

Importing Specific Items

main.mx 
import { add, pi } from "lib/math.mx"

let sum = add(10, 20)
print("Sum: " + sum)
print("Pi: " + pi)

Importing All

main.mx 
import "lib/math.mx"

# All exports are now available
let sum = add(10, 20)

Docs / TensorFlow.js Bridge

TensorFlow.js Bridge

MEX skills transfer directly to TensorFlow.js.

Generate TF.js Code

terminal 
node mex.js --generate my_model.mx

Side-by-Side Comparison

MEX
data points
  (1, 2)
  (2, 4)

model simple
train 100 epochs
predict 5
TensorFlow.js
const xs = tf.tensor2d([[1],[2]]);
const ys = tf.tensor2d([[2],[4]]);

const model = tf.sequential();
model.add(tf.layers.dense({
  units: 1,
  activation: 'linear'
}));

await model.fit(xs, ys, {epochs: 100});
model.predict(tf.tensor2d([[5]]));
🔗 Skill Transfer

The concepts are identical: data, models, training, prediction. MEX uses simpler syntax so you can focus on understanding before implementation.
